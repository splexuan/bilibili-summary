技术说明

整体架构

  • 后端：Python Flask + SQLite，全流程 SSE 流式推送
  • 前端：纯原生 HTML/CSS/JS，无框架依赖，深色模式
  • 支持类型：B站视频、YouTube 视频、文章文本

视频处理流程（4 步）

  1. 解析视频信息 — 通过 yt-dlp 获取标题、UP主、时长、封面，自动识别平台。
  2. 获取文字 — 优先提取视频字幕（B站 AI 字幕 / YouTube 自动字幕），无字幕时下载音频（m4a → ffmpeg 转 16kHz WAV）。完成后自动删除音频，仅保留文字。
  3. 语音转文字 — 使用 sherpa-onnx 加载 SenseVoice Small 模型（INT8 量化），本地 CPU 多线程并行解码（30 秒/块），无需联网。有字幕时自动跳过。
  4. AI 智能总结 — 调用 DeepSeek API（deepseek-v4-flash），流式输出逐字实时显示。

文章总结（1 步直达）

  • 粘贴文章正文 → 直接调用 AI 总结，跳过解析/下载/转写
  • 标题留空时 AI 自动生成（仅消耗约 300 token）
  • 可选填原文链接，总结完成后点击标题可跳转原文

Map-Reduce 长文本总结

  • 直接总结：≤12000 字，一次 API 调用完成。Prompt 要求按"核心概括 → 分点列观点 → 原话摘录"结构输出 Markdown。
  • 二级 Map-Reduce：12000~20000 字，按 语义断句 拆分（非机械切分），每段 ~6500 字，段间重叠 600 字避免关键信息在边界被截断。每段独立调用 AI 提取"概要 + 观点 + 原话"，最后汇总所有段提取的结果整合为完整总结。
  • 三级 Map-Reduce：20000 字以上，每 5 段先归并为一组摘要，再将各组摘要汇总为最终结果。避免超长文本超出 AI 上下文窗口。
  • 前端通过 SSE 实时显示进度（第 N 段/共 M 段 + 每段完成提示），避免长视频处理时的焦虑等待。

单视频/文章 RAG 问答

  • 段落切分：将文本优先按空行分割，段落过少时按句号补充切分，保证每段 ≥10 字。避免过短或过长的段落影响检索效果。
  • 向量化：使用 scikit-learn TfidfVectorizer，字符级 ngram (2-4) 分词。相比词级分词，字符级对中文更友好——不依赖分词准确率，直接捕捉字符组合模式。
  • 检索：用户提问同样做 TF-IDF 向量化，计算与所有段落的 余弦相似度，取 Top 5 最相关段落。仅将相关段落 + 总结作为上下文发送给 AI。
  • 性能优化：索引持久化到 joblib 文件（带 SHA256 校验防止损坏），重启后直接加载。内存 LRU 缓存最多 16 个视频/文章（基于指纹检测文本变更）。

跨视频/文章知识库检索

  • 混合排序：对所有已总结的视频和文章统一建立持久化索引。使用 BM25 + TF-IDF 余弦 双路召回，各 50% 权重合并排序。BM25 擅长"包含用户提问的关键词"，TF-IDF 擅长"语义相近但措辞不同"。
  • 两级检索：第一步在总结文本上做混合排序（粗筛），取出 Top 3 个最相关的视频/文章。第二步对这 3 个的原文/转写做 RAG 精查，找出具体段落。两层过滤确保既不漏掉相关内容、又不浪费 token。
  • 索引更新：持久化到 joblib，仅在总结文件数量或内容变更时自动重建。通过哈希快照对比检测是否需要重建，避免每次请求都重建索引。
  • 同优先级混排：视频和文章在同一索引中按相关性统一排序，来源标签可区分类型，点击跳转对应详情。

存储方案

  • SQLite 统一存储（output/data.db）：videos + articles + chats 三张表
  • 视频封面缓存在 output/thumbnails/，B站封面服务端代理（防盗链）
  • 重复处理时已有转写/总结自动跳过，可手动重新总结

历史记录

  • 📺 视频 / 📄 文章 双标签切换，弹窗固定尺寸不缩放
  • 支持搜索、滑动无限加载、单击加载详情、一键删除
  • 自动回填原始链接和内容，加载文章时自动切到文章模式
  • 重新总结不重新下载和转写

辅助功能

  • 导出 Markdown — 总结一键导出为 .md 文件，含标题和元数据
  • 朗读总结 — Edge TTS，自动去除 Markdown 格式，5 种音色
  • 对话记录 — 视频/文章内对话和知识库全局对话均持久化，支持多轮切换
  • 播放原视频 — 点击封面弹出 B站/YouTube 内嵌播放器
  • 任务并发控制 — 最多 3 并行，API 限流 10 次/60 秒

下载与网络

  • B站未登录限速 ~300KB/s，设置中粘贴 Cookie 即时生效
  • 支持 HTTP / SOCKS 代理
  • yt-dlp 以 Python 模块方式调用（python -m yt_dlp）

隐私说明

  • 语音识别 完全本地运行（sherpa-onnx），不上传音频
  • 仅在 AI 总结/对话时，文字内容发送到 DeepSeek API
  • Key 保存在 ~/.bilibili-summary-key，Cookie 在 cookies.txt
